# 秒杀服务

### 秒杀（高并发）系统关注问题
- 服务单一职责，独立部署
    秒杀服务即使自己扛不住压力，挂掉，也不会影响其他
- 秒杀连接加密
    防止恶意攻击，模拟秒杀请求，防止链接暴露，提前秒杀商品
- 库存预热+快速扣减
    秒杀要读多写少，无需每次实时校验库存，我们库存预热，放到redis中，信号量控制秒杀请求
- 动静分离
    使用cdn
- 恶意请求拦截
    获取非法攻击请求进行拦截，网关层实现
- 流量错峰
    使用各种手段，将流量分担到更大宽度的时间点，比如验证码，加入购物车
- 限流&熔断&降级
    前端限流 + 后端限流
    限制次数，限制总量，快速失败降级运行，熔断隔离防止雪崩
- 队列削峰
    所有秒杀成功的请求，进入队列，慢慢创建订单，扣减库存即可
